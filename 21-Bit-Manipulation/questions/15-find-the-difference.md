# Q15 — Find the Difference

**Difficulty:** Medium
**Pattern:** XOR on character codes
**Expected:** O(n) time · O(1) space

## Problem

You are given two strings `s` and `t`, where `t` is generated by randomly shuffling string `s` and then adding **one additional random character** at a random position.

Return the letter that was added to `t`.

## Examples

### Example 1
```
Input:  s = "abcd", t = "abcde"
Output: 'e'
```

### Example 2
```
Input:  s = "", t = "y"
Output: 'y'
```

### Example 3
```
Input:  s = "a", t = "aa"
Output: 'a'
```

## Constraints
- `0 <= s.length <= 1000`
- `t.length == s.length + 1`
- `s` and `t` consist of lowercase English letters.

## Hints

<details>
<summary>Hint 1 — it's the same idea as Single Number (Q11)</summary>

Every character in `s` also appears in `t` — each character from `s` is paired with itself in `t`. The extra character in `t` has no pair.

XOR all character codes from `s` and `t` together. Pairs cancel. The extra character's code remains.
</details>

<details>
<summary>Hint 2 — getting a character's code</summary>

Use `str.charCodeAt(i)` to get the numeric ASCII value of character i.
Use `String.fromCharCode(code)` to convert back to a character.

```js
'a'.charCodeAt(0)   // 97
String.fromCharCode(97)  // 'a'
```
</details>

<details>
<summary>Hint 3 — implementation sketch</summary>

```js
let xor = 0;
for (const c of s) xor ^= c.charCodeAt(0);
for (const c of t) xor ^= c.charCodeAt(0);
return String.fromCharCode(xor);
```

Because every character in `s` appears exactly once in both loops (via t's shuffle), they cancel. The extra character only appears in the second loop, so it survives.
</details>
