# Q35 — 24 Game

**Difficulty:** Hard
**Pattern:** Backtracking — try all pairs, all operators, all parenthesizations recursively
**Expected:** O(1) time (input is always 4 numbers) · O(1) space

## Problem

You are given an integer array `cards` of length 4. You must evaluate whether you can produce the value `24` by applying the four arithmetic operators `+`, `-`, `*`, `/` to the four numbers. Each number must be used **exactly once**, and you may arrange them in any order and group them with parentheses in any way.

Division is real-valued (not integer division). An expression like `8 / (3 - 8/3)` is valid.

Return `true` if it is possible to obtain `24`, otherwise `false`.

## Examples

### Example 1
```
Input:  cards = [4, 1, 8, 7]
Output: true
```
`(8 - 4) * (7 - 1) = 4 * 6 = 24`

### Example 2
```
Input:  cards = [1, 2, 1, 2]
Output: false
```
No arrangement of `1, 1, 2, 2` with `+, -, *, /` evaluates to 24.

## Constraints
- `cards.length == 4`
- `1 <= cards[i] <= 9`

## Hints

<details>
<summary>Hint 1 — recursive reduction</summary>

At each step, pick any two numbers from the remaining list, apply one of the four operators to produce a new number, and recurse with the resulting list of `n - 1` numbers. When only one number remains, check if it equals 24 (within floating-point tolerance).
</details>

<details>
<summary>Hint 2 — all operator orderings are covered by recursion</summary>

You don't need to enumerate parenthesizations explicitly. The recursive "pick two, combine, repeat" approach naturally covers all binary tree structures (i.e., all parenthesizations) because you choose which two numbers to combine at each depth level.
</details>

<details>
<summary>Hint 3 — floating-point comparison</summary>

Due to floating-point errors, compare the final result against 24 with a small epsilon: `Math.abs(value - 24) < 1e-6`. Also handle division by zero: if the divisor is approximately zero, skip that operator combination.
</details>
