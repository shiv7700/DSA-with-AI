# 01 — Big-O & Complexity Analysis

> Big-O is the language engineers use to talk about how algorithms scale. Master it once and you'll have a permanent lens for evaluating every data structure and algorithm you ever encounter.

## Learn Path

1. **Read [`notes.md`](./notes.md) first** — 18 lessons covering every complexity class, the analysis method, JS pitfalls, space complexity, and amortized analysis.
2. Work through the questions in order. Conceptual questions (01–05, 09–14) train your reasoning. Snippet questions (06–13) train your eye. Optimization questions (15–17) train your instinct.
3. Write your answers in the matching `solutions/NN-slug.js` (as comments for conceptual questions, as actual code for optimization questions).
4. Tick off the checklist below as you finish.
