# Q10 — Extra Character Between Two Strings

**Difficulty:** Easy
**Pattern:** Frequency map difference
**Expected:** O(n) time · O(k) space

## Problem

You are given two strings `s` and `t`. String `t` is generated by taking `s`, inserting **exactly one** extra character at a random position, and shuffling the result. Find and return that extra character.

## Examples

### Example 1
```
Input:  s = "abcd",  t = "abecd"
Output: 'e'
```

### Example 2
```
Input:  s = "aa",  t = "aaa"
Output: 'a'
```

### Example 3
```
Input:  s = "",  t = "z"
Output: 'z'
```

### Example 4
```
Input:  s = "abcde",  t = "edbcfa"
Output: 'f'
```

## Constraints
- `0 <= s.length <= 10^3`
- `t.length == s.length + 1`
- `s` and `t` consist of lowercase English letters.

## Hints

<details>
<summary>Hint 1 — frequency map approach</summary>

Build a frequency map of all characters in `s`. Then walk through `t`, decrementing each character's count. The character whose count drops to `-1` (or whose count is `0` when you try to decrement) is the extra one.
</details>

<details>
<summary>Hint 2 — XOR trick (bonus approach, O(1) space)</summary>

XOR has a useful property: `x ^ x = 0` and `x ^ 0 = x`. XOR all characters in `s` and `t` together. Every paired character cancels out, leaving only the extra character.

```js
function findExtraChar(s, t) {
  let result = 0;
  for (const c of s) result ^= c.charCodeAt(0);
  for (const c of t) result ^= c.charCodeAt(0);
  return String.fromCharCode(result);
}
```

Know both approaches — the frequency-map version is what belongs in this chapter; the XOR version is a clever O(1) space trick.
</details>
